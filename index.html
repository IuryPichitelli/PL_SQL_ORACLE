Tópico 01

Oque é um dado?

    Um dado é um conhecimento que temos sobre alguma coisa.

    Com esse conhecimento, chegamos a uma conclusão. E com essa conclusão, tomamos alguma ação.

O que é um Banco de Dados?

    Um banco de dados é um local onde você guarda informações de forma organizada, como se fosse uma estante cheia de pastas bem arrumadas. Ele serve para:

    • Armazenar dados (nomes, números, registros etc.).
    • Organizar essas informações.
    • Permitir buscas rápidas quando você precisa encontrar algo.

Oque é um banco de dados relacional?

    Um banco de dados relacional é um tipo de banco de dados que guarda as informações em tabelas (linhas e colunas), como em uma planilha do Excel. O “relacional” vem do fato de que essas tabelas podem estar ligadas entre si por meio de chaves (IDs).

    Exemplo:

    • Tabela Clientes (id, nome, telefone)
    • Tabela Pedidos (id, cliente_id, produto)

    O campo cliente_id conecta um pedido a um cliente específico.

    Ou seja: é um banco que organiza dados em tabelas e permite relacioná-las para evitar bagunça e repetição.

Exemplos de SGBDs

  • MySQL
  • Oracle
  • SQL Server
  • PostgreSQL

Tópico 02

Oque é SQL?

    SQL (Structured Query Language) é uma linguagem usada para conversar com bancos de dados relacionais.

      Com ela você pode:

    • Criar tabelas e bancos de dados.
    • Inserir dados (adicionar registros).
    • Consultar informações (buscar o que precisa).
    • Atualizar dados já existentes.
    • Excluir dados ou tabelas.

Oque é uma QUERY (consulta) ?

    Uma query é um comando que você dá para o banco de dados, pedindo para buscar, adicionar, mudar ou apagar informações.

Tópico 03

Termos importantes:

    No SQL existem alguns termos (ou comandos) que aparecem o tempo todo e são os mais importantes de aprender primeiro. Aqui estão os principais:

    • SELECT → usado para buscar dados em uma tabela.
    • FROM → diz de qual tabela os dados vão ser buscados.
    • WHERE → coloca uma condição (ex.: “só os clientes da cidade X”).
    • INSERT → para inserir dados novos.
    • UPDATE → para atualizar dados existentes.
    • DELETE → para apagar dados.
    • CREATE → cria tabelas ou bancos de dados.
    • DROP → apaga tabelas ou bancos inteiros.
    • JOIN → conecta dados de duas ou mais tabelas.
    • ORDER BY → organiza os resultados em ordem crescente ou decrescente.
    • GROUP BY → agrupa registros (muito usado com funções como COUNT, SUM, AVG).

Tópico 04

SQL vs MySQL, SQL SERVER, ORACLE e POSTGRESQL

SQL

  • É uma linguagem.
  • Serve para conversar com bancos de dados relacionais.
  • É como o “português” usado para se comunicar.

MySQL, SQL SERVER, ORACLE e POSTGRESQL

  • São SGBDs (Sistemas de Gerenciamento de Bancos de Dados).
  • Cada um é um software que usa o SQL (com algumas variações) para armazenar, organizar e acessar dados.
  • É como ter diferentes aplicativos que entendem a mesma língua, mas cada um com seus recursos extras.

    • Diferenças entre eles de forma resumida:

    • MySQL → Muito usado em sites e aplicações web. É gratuito (open source) e simples de configurar.
    • SQL Server → Da Microsoft. Muito usado em empresas que trabalham com o ecossistema Windows.
    • Oracle Database → Da Oracle. Muito poderoso e robusto, comum em grandes corporações.
    • PostgreSQL → Também gratuito e open source. Conhecido por ser bem completo e seguir mais fielmente o padrão SQL.

Tópico 05

Por que aprender SQL?

    1.É essencial para trabalhar com dados

  • Bancos de dados estão em tudo: sites, apps, empresas, jogos, e-commerce.
  • Saber SQL te permite buscar, organizar e analisar informações.

    • 2. Habilidade valorizada no mercado

  • Profissionais que sabem SQL são procurados em TI, análise de dados, marketing, finanças e mais.
  • Serve como base para aprender ferramentas de BI, Big Data e Data Science.

    • 3. É a “linguagem universal” dos bancos de dados relacionais

  • Mesmo que mude o software (MySQL, PostgreSQL, Oracle…), o SQL continua quase o mesmo.
  • Ou seja, você aprende uma vez e usa em vários lugares.

    • 4. Ajuda a entender como os sistemas funcionam

  • Aprender SQL faz você entender melhor como sites e aplicativos armazenam e recuperam informações.

Criando usuário SYS

    Para criar uma conexão como o usuário admin do sistema, teremos que colocar as seguintes configurações:

  • Name: ADMIN
  • Username: sys (usuário padrão do sistema)
  • Password: senha cadastrada ao instalar o DB
  • Role: SYSDBA (Nível mais alto de permissão de uma banco de dados = modo deus)

      • SYSDBA permite:

      • Criar ou apagar bancos de dados.
      • Executar backup e recovery.
      • Gerenciar usuários e permissões sem restrições.
      • Abrir/montar/desmontar o banco (STARTUP e SHUTDOWN).
  • Service name: XEPDB1 (eXpress Edition Pluggable Database 1)

      • Explicação:

      • No Oracle XE (Express Edition), quando você instala o banco de dados, ele cria automaticamente alguns containers e pluggable databases (PDBs).
      • CDB (Container Database): É o “banco de dados principal” que contém informações de controle e metadados dos bancos que podem ser conectados a ele.
      • PDB (Pluggable Database): É um “banco de dados plugável”, que funciona como um banco de dados separado dentro do CDB.
      • No Oracle XE 18c/21c:
        • XEPDB1 é o pluggable database padrão criado automaticamente durante a instalação.
        • Ele é o PDB que você geralmente conecta para criar tabelas, usuários e executar queries.
        • Resumindo: XEPDB1 é o nome do banco de dados “pronto para usar” dentro do Oracle XE. É nele que você vai trabalhar normalmente.
  • Test: Clicando em testar, o DB vai testar se todos os inputs colocados foram corretos.
  • Status: Sucess: essa é a mensagem que vai aparecer se tudo estiver correto.

Criando os próprios usuários

    Para criarmos o próprio usuário, devemos acessar a Oracle Connections > ADMIN, dentro do DB vamos percorrer todas as pastas até chegar na última chamada Other Users, nela iremos clicar com o botão direito em cima da pasta Other Users e clicaremos na opção Create User.., dentro do formulário Create User iremos colocar seu username seguido de uma senha. Após criarmos o nome e senha do usuário iremos em Granted Roles, aonde podemos atribuir ao usuário criado alguns pacotes de permissões do DB, e logo após por exemplo atribuirmos ao usuário a sua role podemos também atribuir mais outras grant, porém não podemos atribuir todas as grant, pois estariamos dando ao usuário o mesmo nível de permissão que o ADMIN oque não é conveniente. Por fim o sistema não permitirá que o usuário tenha os grant de nível ADMIN, assim o sistema atruibuirá algumas grant padrões, oque por sua vez pode ser alterada.

    O que é um role?
    Um role é um conjunto de privilégios (permissões) agrupados em um só “pacote”. Em vez de dar cada privilégio individualmente a um usuário, você cria um role e depois o concede ao usuário.

Por fim colocaremos o service name, e clicaremos em connect e pronto, conexão criada no DB.

Dúvidas:

  • Oque é XEPDB1?

      • O XEPDB1 é o nome padrão de um “pluggable database” (PDB) do Oracle Database Express Edition (XE).

  • Conceitos básicos do Oracle moderno:

      • O Oracle 12c+ mudou o modelo de banco de dados:

    • CDB (Container Database) → é o “banco principal” que contém a infraestrutura.
    • PDB (Pluggable Database) → são “bancos menores” que ficam dentro do CDB. Você pode ter vários PDBs dentro de um CDB.
      • Pense no CDB como um prédio e os PDBs como apartamentos. Cada apartamento é independente, mas todos compartilham a mesma estrutura do prédio.
  • No Oracle XE
    • O XE vem com um CDB chamado XE.
    • Ele cria automaticamente um PDB chamado XEPDB1.
    • Então quando você instala o Oracle XE, você já tem um banco pronto para usar, que é o XEPDB1.
  • Para que serve?
    • É o banco que você vai usar para criar tabelas, inserir dados, fazer consultas, etc.
    • No SQL Developer, você se conecta a ele usando o service name XEPDB1.

CRUD ( Comandos DDL, DML e TCL )

  • O que é CRUD e o que são os comandos DDL e DML?

      • CRUD é um nome dado para as operações realizadas dentro de um banco de dados. É a sigla para: CREATE, READ, UPDATE e DELETE.

      De forma mais geral, podemos dividir os comandos dentro de um banco de dados em alguns grupos:

      • DDL: Data Definition Language = São comandos para manipulação de objetos em um banco de dados.

          • Exemplos: CREATE, ALTER, TRUNCATE, DROP, COMMENT

      • DML: Data Manipulation Language = São comandos para manipulação de dados em um banco de dados.

          • Exemplos: INSERT, UPDATE, DELETE, SELECT

  • Tipos de Dados:

      • Os dados armazenados dentro de um banco de dados poderão ser de uma série de tipos.
      Abaixo, listamos os mais comuns:

      1. INT
      2. NUMBER ( p, s )
      3. VARCHAR2 ( n )
      4. DATE

      Aprofundando:

          Tipos de caracteres:

          Usados para armazenar dados de texto.

        • CHAR (size) : Armazena uma string de caracteres de tamanho fixo. Ocupa sempre a quantidade de bytes definida em size, mesmo que o texto armazenado seja menor.
        • VARCHAR2 (size) : Armazena uma string de caracteres de tamanho variável. Ocupa apenas o espaço necessário para o texto, até o limite máximo definido em size. É o tipo de texto mais comum no Oracle.
        • NCHAR (size) e NVARCHAR2 (size) : Versões dos tipos CHAR e VARCHAR2 que suportam conjuntos de caracteres nacionais, como o Unicode, para armazenar dados em vários idiomas.
        • CLOB : Usado para armazenar grandes blocos de dados de caracteres, como textos longos ou documentos.

          Tipos numéricos

          Usados para armazenar valores numéricos.

        • NUMBER (p, s) : Um tipo numérico com alta precisão e escala configuráveis. p é a precisão (o número total de dígitos) e s é a escala (o número de dígitos à direita da vírgula decimal).
        • NUMBER : Quando a precisão e a escala não são especificadas, o Oracle as ajusta dinamicamente, permitindo armazenar números inteiros ou decimais.
        • INTEGER : Usado para armazenar números inteiros.
        • FLOAT : Usado para armazenar valores numéricos aproximados, de ponto flutuante.
        • BINARY_FLOAT e BINARY_DOUBLE : Tipos numéricos que armazenam valores de ponto flutuante com base nas normas IEEE 754, sendo mais eficientes para cálculos que requerem alta performance.

          Tipos de data e hora

          Usados para armazenar informações temporais.

        • DATE : Armazena a data (dia, mês e ano) e a hora (hora, minuto e segundo).
        • TIMESTAMP : Oferece maior precisão que o DATE, armazenando também frações de segundo.

          Tipos binários

          Usados para armazenar dados binários, como imagens, áudio ou arquivos.

        • RAW (size) : Usado para armazenar dados binários de tamanho variável, de forma similar ao VARCHAR2, mas para dados não textuais.
        • BLOB : Armazena grandes objetos binários, como imagens ou arquivos.
        • BFILE : Armazena um ponteiro para um arquivo binário que está fora do banco de dados, no sistema de arquivos do servidor.

          Obs:
          No VALUES data_nascimento DATE tivemos que usar o TO_DATE pelos seguinte motivo:

          • Quando você insere uma data como string ('11/10/2001'), o banco tenta converter automaticamente usando o formato de data padrão da sua sessão (NLS_DATE_FORMAT). Se o NLS_DATE_FORMAT estiver como YYYY-MM-DD, o Oracle espera '2001-10-11'.

            • - Regra especial do DD-MM-RRRR (Diferença)

            O Oracle inventou o RR para facilitar a transição do ano 2000:

            • Se você usar YY, ele pega os 2 dígitos e completa com o século atual.
            • Se você usar RR, ele tenta adivinhar o século com base na data atual.
            • Exemplo (supondo que hoje seja ano 2025):
              • sql

              • TO_DATE( '49-12-15','RR-MM-DD') --> 2049-12-15 (ano 2049)

              • TO_DATE('50-12-15','RR-MM-DD') --> 1950-12-15 (ano 1950)

              • TO_DATE('25-12-15','YY-MM-DD') --> 2025-12-15 (ano 2025)

            Dica:

          • Mesmo que você saiba o formato, é boa prática sempre usar TO_DATE com o formato explícito (DD/MM/YYYY, YYYY-MM-DD, etc.), porque assim seu código funciona em qualquer ambiente, independente da configuração regional.

FILTROS AVANÇADOS

    Comando Descrição Exemplo
    * Termo para selecionar todas as colunas de uma tabela SELECT * FROM funcionarios;
    SELECT Seleciona a coluna que queremos SELECT * FROM funcionarios;
    FROM Usado para referenciar qual tabela iremos extrair os dados SELECT * FROM funcionarios;
    WHERE Condição (ONDE) que podemos usar para colocar condições na pesquisa SELECT * FROM funcionarios WHERE id = 2;
    LIKE Usado para procurar qualquer sequência sendo ela numerica ou alfabetica SELECT * FROM funcionarios WHERE name_func LIKE 'Pedro%';
    LIKE 'x%' Sequência aonde os caracteres a direita da palavra podem ser qualquer um SELECT * FROM funcionarios WHERE name_func LIKE 'Pedro%';
    LIKE 'x_' Procura por um unico caracter que começe com ele SELECT * FROM funcionarios WHERE name_func LIKE 'Pedro_';
    NOT LIKE 'x_' ou 'x%' Tem a função de negação, tudo que não estiver dentro do parâmetros do LIKE SELECT * FROM funcionarios WHERE name_func NOT LIKE 'Pedro_';
    BETWEEN ... AND Uma atribuição que usa dois valores e puxa oque está entre eles. SELECT * FROM funcionarios WHERE year BETWEEN 2000 AND 2010;
    NOT BETWEEN … AND Uma atribuição que usa dois valores e puxa oque NÃO está entre eles. SELECT * FROM funcionarios WHERE year NOT BETWEEN 2000 AND 2010;
    IN (…) (inclusive / ou) Usado para retornar os valores, podendo ser tanto um ou outro dentro da lista SELECT * FROM funcionarios WHERE sexo IN ('M','F');
    NOT IN (…) (não inclusive / ou) Usado para retornar os valores, podendo ser tanto um ou outro dentro da lista SELECT * FROM funcionarios WHERE name_func NOT IN ('João','Fernanda');
    ORDER BY ... ASC / DESC Usado para ordernar uma coluna, sendo ela numericamente ou alfabeticamente. Sendo ASC = crescente e DESC = decrescente. SELECT * FROM funcionarios ORDER BY qtd_funcionarios ASC ;
    DISTINCT Usado para remover cegamente palavras duplicadas. Também funciona com números, datas, etc.. SELECT DISTINCT nome_funcionarios FROM funcionarios;
    LIMIT Usado para reduzir o número de linhas a serem retornadas SELECT * FROM funcionarios LIMIT 5;
    OFFSET Usado para especificar aonde começará a contagem do número de linhas. SELECT * FROM funcionarios OFFSET 5;
    INNER JOIN & ON Usado para fazer junções de duas tabelas em uma pesquisa.Cláusula em uma consulta, podemos combinar dados de linhas de duas tabelas separadas usando essa chave exclusiva. SELECT * FROM movies INNER JOIN boxoffice ON id = movie_id
    GROUP BY É usado para agrupar linhas de uma tabela que têm valores iguais em uma ou mais colunas. SELECT movies_name, SUM(cost_movies) AS total_salario FROM movies GROUP BY movies_name
    GROUP BY (AGREGAÇÃO) COUNT() → conta registros , SUM() → soma valores, AVG() → média, MAX() → maior valor, MIN() → menor valor
