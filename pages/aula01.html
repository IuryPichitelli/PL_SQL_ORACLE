Tópico 01

Oque é um dado?

    Um dado é um conhecimento que temos sobre alguma coisa.

    Com esse conhecimento, chegamos a uma conclusão. E com essa conclusão, tomamos alguma ação.

O que é um Banco de Dados?

    Um banco de dados é um local onde você guarda informações de forma organizada, como se fosse uma estante cheia de pastas bem arrumadas. Ele serve para:

    • Armazenar dados (nomes, números, registros etc.).
    • Organizar essas informações.
    • Permitir buscas rápidas quando você precisa encontrar algo.

Oque é um banco de dados relacional?

    Um banco de dados relacional é um tipo de banco de dados que guarda as informações em tabelas (linhas e colunas), como em uma planilha do Excel. O “relacional” vem do fato de que essas tabelas podem estar ligadas entre si por meio de chaves (IDs).

    Exemplo:

    • Tabela Clientes (id, nome, telefone)
    • Tabela Pedidos (id, cliente_id, produto)

    O campo cliente_id conecta um pedido a um cliente específico.

    Ou seja: é um banco que organiza dados em tabelas e permite relacioná-las para evitar bagunça e repetição.

Exemplos de SGBDs

  • MySQL
  • Oracle
  • SQL Server
  • PostgreSQL

Tópico 02

Oque é SQL?

    SQL (Structured Query Language) é uma linguagem usada para conversar com bancos de dados relacionais.

      Com ela você pode:

    • Criar tabelas e bancos de dados.
    • Inserir dados (adicionar registros).
    • Consultar informações (buscar o que precisa).
    • Atualizar dados já existentes.
    • Excluir dados ou tabelas.

Oque é uma QUERY (consulta) ?

    Uma query é um comando que você dá para o banco de dados, pedindo para buscar, adicionar, mudar ou apagar informações.

Tópico 03

Termos importantes:

    No SQL existem alguns termos (ou comandos) que aparecem o tempo todo e são os mais importantes de aprender primeiro. Aqui estão os principais:

    • SELECT → usado para buscar dados em uma tabela.
    • FROM → diz de qual tabela os dados vão ser buscados.
    • WHERE → coloca uma condição (ex.: “só os clientes da cidade X”).
    • INSERT → para inserir dados novos.
    • UPDATE → para atualizar dados existentes.
    • DELETE → para apagar dados.
    • CREATE → cria tabelas ou bancos de dados.
    • DROP → apaga tabelas ou bancos inteiros.
    • JOIN → conecta dados de duas ou mais tabelas.
    • ORDER BY → organiza os resultados em ordem crescente ou decrescente.
    • GROUP BY → agrupa registros (muito usado com funções como COUNT, SUM, AVG).

Tópico 04

SQL vs MySQL, SQL SERVER, ORACLE e POSTGRESQL

SQL

  • É uma linguagem.
  • Serve para conversar com bancos de dados relacionais.
  • É como o “português” usado para se comunicar.

MySQL, SQL SERVER, ORACLE e POSTGRESQL

  • São SGBDs (Sistemas de Gerenciamento de Bancos de Dados).
  • Cada um é um software que usa o SQL (com algumas variações) para armazenar, organizar e acessar dados.
  • É como ter diferentes aplicativos que entendem a mesma língua, mas cada um com seus recursos extras.

    • Diferenças entre eles de forma resumida:

    • MySQL → Muito usado em sites e aplicações web. É gratuito (open source) e simples de configurar.
    • SQL Server → Da Microsoft. Muito usado em empresas que trabalham com o ecossistema Windows.
    • Oracle Database → Da Oracle. Muito poderoso e robusto, comum em grandes corporações.
    • PostgreSQL → Também gratuito e open source. Conhecido por ser bem completo e seguir mais fielmente o padrão SQL.

Tópico 05

Por que aprender SQL?

    1. É essencial para trabalhar com dados

  • Bancos de dados estão em tudo: sites, apps, empresas, jogos, e-commerce.
  • Saber SQL te permite buscar, organizar e analisar informações.

    • 2. Habilidade valorizada no mercado

  • Profissionais que sabem SQL são procurados em TI, análise de dados, marketing, finanças e mais.
  • Serve como base para aprender ferramentas de BI, Big Data e Data Science.

    • 3. É a “linguagem universal” dos bancos de dados relacionais

  • Mesmo que mude o software (MySQL, PostgreSQL, Oracle…), o SQL continua quase o mesmo.
  • Ou seja, você aprende uma vez e usa em vários lugares.

    • 4. Ajuda a entender como os sistemas funcionam

  • Aprender SQL faz você entender melhor como sites e aplicativos armazenam e recuperam informações.
